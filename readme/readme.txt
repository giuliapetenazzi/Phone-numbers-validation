South-African mobile phone number validation

Il progetto fa riferimento alla consegna indicata nel file:
Pre-selezione. Challenge_Backend_South_African_Numbers_.pdf

Server files:
  ./server.js

Client files:
  ./views/index.ejs
  ./public/css/styles.css/
  ./public/css/script.js

Test files:
  ./test/unitTestClient.js
  ./test/unitTestServer.js

Configurazione e dipendenze:
  fare riferimento al file package.json

Per far avviare il progetto, una volta installate le dipendenze:
  npm start

Per far partire i test, una volta installate le dipendenze:
  mocha  

Backe-end:
  NPM, Express, ES6 [...]
Front-end:
  HTML, CSS3, Bootstrap, ES6 [...]
  
Note sulla consegna:
  I numeri telefonici sono corretti se composti da un 27 seguito da 9 cifre decimali.
  I numeri correggibili sono quelli composti da esattamente 9 cifre decimali, anteponendo il 27.
  Tutti gli altri numeri non saranno considerati validi.
